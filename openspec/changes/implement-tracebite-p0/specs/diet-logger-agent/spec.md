## ADDED Requirements

### Requirement: 意圖理解與分流
`DietLoggerAgent` MUST 能夠識別使用者輸入的意圖，並正確執行對應的分流：
1. 若意圖為「新增今日飲食紀錄（含照片或文字描述）」，則呼叫圖片分析、營養查詢與資料儲存工具。
2. 若意圖為「查詢今日飲食摘要」，則呼叫資料庫讀取工具獲取今日數據。
3. 若意圖為「查詢本週飲食摘要」，則呼叫資料庫讀取工具獲取本週數據。

#### Scenario: 成功識別新增紀錄意圖
- **WHEN** 使用者輸入包含今日餐點照片並指定餐別為「午餐」
- **THEN** Agent 呼叫圖片辨識工具進行食物項目分析

#### Scenario: 成功識別今日摘要查詢意圖
- **WHEN** 使用者詢問「今天吃了多少熱量？」
- **THEN** Agent 呼叫資料庫讀取工具查詢今日的飲食摘要並回覆

---

### Requirement: 資料來源可追溯與警示回覆
Agent 回覆使用者的飲食分析結果或摘要時，SHALL 包含資料來源的唯讀記錄（例如 `readonly_sources`），且必須附帶免責聲明。

#### Scenario: 顯示免責聲明與資料來源
- **WHEN** Agent 完成飲食紀錄並生成回覆
- **THEN** 回覆內容中包含「此結果為一般飲食紀錄與營養估算，不取代醫師或營養師建議。」之文字，且詳細條列偵測來源

---

### Requirement: 關鍵函數執行日誌紀錄
Agent 內部之關鍵函數（包括意圖判斷、工具呼叫、圖片分析工具、營養估算工具與資料庫操作）SHALL 在其執行起點、轉折點及終點寫入詳細日誌，並寫入本地日誌檔案以供追蹤排錯。

#### Scenario: 成功記錄工具呼叫日誌
- **WHEN** Agent 決定呼叫 `estimate_nutrition` 工具
- **THEN** 系統於本地日誌檔寫入工具呼叫名稱、傳入參數以及工具返回之估算結果與來源資料
